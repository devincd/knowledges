### 概述
为什么选择分析源码包？
- 标准库是 go 的核心
- 标准库的使用是必修课，既然要学为何不直接知其所以然
- 站在巨人的肩膀上，「学得多看得远」

我认为通过标准库源码的分析可以有以下的收获：
- 标准库的使用
- 标准库的底层实现
- 学习大神的编码哲学
- 极大的提高编码能力和水平
- 提升英文水平「源码注释都是英文的」

> 能看懂内核里面的代码就是最大的幸福，更不用说修改里面的代码了

### 如何分析
整体将按照如下思路进行
1. 整理出所有的包以及每个包大概的用途
2. 以围绕**fmt**包展开
3. 首先展开**fmt**包的依赖，然后对依赖进行逐一分析「类似于执行 fmt 包的加载过程一样」

如何对包进行分析
- 翻译注释
- 解读源码分析逻辑(分析包中所有公开的方法和核心结构体)
- 思考和练习包的使用
- 如果可以自己尝试实现包
- 总结
